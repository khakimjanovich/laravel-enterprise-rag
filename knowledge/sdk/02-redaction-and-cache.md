## Rule: All Log Output Must Pass Through Redactor

The SDK never logs raw request or response data.
Every log call passes payload and headers through `Redactor::redact()` first.

**Correct — svgate ApiCaller:**
```php
$this->logger->debug('svgate.request.created', [
    'method' => $method,
    'request_id' => $requestId,
    'headers' => $this->redactor->redact($headers),  // ← redacted
    'payload' => $this->redactor->redact($payload),  // ← redacted
]);

$this->logger->debug('svgate.response.received', [
    'method' => $method,
    'request_id' => $requestId,
    'status_code' => $response->statusCode,
    'result' => $this->redactor->redact($decoded['result']), // ← redacted
]);
```

**Forbidden:**
```php
$this->logger->debug('request', ['payload' => $payload]); // raw — leaks PAN, token
$this->logger->debug('response', ['result' => $result]);   // raw — leaks card data
```

## Rule: Redactor Lives in Internal/ and Is Never Public

`Redactor` is an internal implementation detail.
It is instantiated inside `ApiCaller`, never injected by the consumer.

**Correct — svgate ApiCaller constructor:**
```php
public function __construct(
    private readonly ClientOptions $options,
    ?RequestIdGenerator $requestIdGenerator = null,
    ?Redactor $redactor = null,              // ← optional for testing only
) {
    $this->redactor = $redactor ?? new Redactor();
}
```

The consumer never sees or references `Redactor` directly.
It is only injectable via the constructor for unit test overrides.

## Rule: Sensitive Keys Are Defined as a Constant List

The redacted field list is a private constant array inside `Redactor`.
It is never configurable by the consumer — the SDK decides what is sensitive.

**Correct — svgate Redactor:**
```php
private const array SENSITIVE_KEYS = [
    'authorization',
    'cardid',
    'code',
    'hpan',
    'pan',
    'pan2',
    'password',
    'phone',
    'pinfl',
    'recipient',
    'requestorphone',
    'sender',
    'token',
];
```

These cover: card identifiers, auth credentials, personal identifiers, phone numbers.

## Rule: Redaction Is Case-Insensitive and Recursive

The `Redactor` normalizes keys to lowercase before comparison.
It recursively redacts nested arrays.

**Correct — svgate Redactor::redact():**
```php
public function redact(array $data): array
{
    $redacted = [];

    foreach ($data as $key => $value) {
        $normalizedKey = is_string($key) ? mb_strtolower($key) : '';

        if (is_array($value)) {
            $redacted[$key] = $this->redact($value); // ← recursive
            continue;
        }

        $redacted[$key] = in_array($normalizedKey, self::SENSITIVE_KEYS, true)
            ? '[REDACTED]'
            : $value;
    }

    return $redacted;
}
```

This means `Authorization`, `AUTHORIZATION`, and `authorization` are all redacted.
Nested objects like `sender.pan` or `credit.recipient` are also redacted.

---

## Rule: Redacted Value Is Always the String [REDACTED]

The replacement value is always the literal string `'[REDACTED]'`.
Never `null`, never `***`, never an empty string.

**Correct:**
```php
$redacted[$key] = '[REDACTED]';
```

**Forbidden:**
```php
$redacted[$key] = null;   // loses the key presence — hides that field existed
$redacted[$key] = '***';  // non-standard — hard to grep in logs
$redacted[$key] = '';     // indistinguishable from an empty real value
```

`[REDACTED]` is grep-friendly, visually obvious, and preserves the key in the log.

## Rule: Request ID Is Generated Once Per Call

Every outbound request gets a unique request ID generated by `RequestIdGenerator`.
The ID is a 32-character hex string from 16 random bytes.

**Correct — svgate RequestIdGenerator:**
```php
final class RequestIdGenerator
{
    public function generate(): string
    {
        return bin2hex(random_bytes(16));
    }
}
```

**Generated format:** `a3f2c1d4e5b6a7f8c9d0e1f2a3b4c5d6` (32 hex chars)

---

## Rule: Request ID Flows Through the Full Call Lifecycle

The request ID is generated once at the start of `ApiCaller::call()` and carried through:
- outbound request log
- JSON-RPC envelope (`id` field)
- response validation (id must match)
- response log
- all warning logs on failure

**Correct — svgate ApiCaller::call():**
```php
public function call(string $method, array $params, ?string $requestId = null): array
{
    $requestId ??= $this->requestIdGenerator->generate(); // ← generated once

    $payload = [
        'jsonrpc' => '2.0',
        'method' => $method,
        'id' => $requestId,          // ← in envelope
        'params' => $params,
    ];

    $this->logger->debug('svgate.request.created', [
        'request_id' => $requestId,  // ← in request log
        ...
    ]);

    // ... transport call ...

    if (! $this->matchesResponseId($decoded['id'] ?? null, $requestId)) {
        throw SVGateException::invalidResponseId(); // ← validated against response
    }

    $this->logger->debug('svgate.response.received', [
        'request_id' => $requestId,  // ← in response log
        ...
    ]);
}
```

This means every request and its response share the same ID in logs.
You can grep logs by request ID to reconstruct a full call trace.

## Rule: Consumer Can Provide Their Own Request ID

The consumer can inject a custom request ID via the Payload.
If none is provided, `RequestIdGenerator` generates one automatically.

**Correct — svgate SVGate::call():**
```php
private function call(BasePayload $payload): array
{
    $params = $payload->toArray();
    $requestId = isset($params['request_id']) && is_scalar($params['request_id'])
        ? (string) $params['request_id']
        : null;           // ← null means ApiCaller generates one

    unset($params['request_id']); // ← never sent as a param to provider

    return $this->caller->call(
        method: $payload->getMethodName(),
        params: $params,
        requestId: $requestId,
    );
}
```

The `request_id` field in Payload is a consumer-facing correlation hook.
It is stripped before the provider call.

## Rule: Transport Failures Are Logged Before Re-throwing

When transport fails, the SDK logs a warning with method and request ID before throwing.

**Correct — svgate ApiCaller:**
```php
} catch (Throwable $exception) {
    $this->logger->warning('svgate.transport.failed', [
        'method' => $method,
        'request_id' => $requestId,
        'exception' => $exception::class,  // ← class name only, not message
    ]);

    throw SVGateException::connection($exception);
}
```

Only the exception class name is logged — never the raw exception message,
which may contain sensitive transport details.

---

## Redaction and Request Identity Checklist

Before committing logging code in an SDK:

- [ ] Every `logger->debug()` call passes data through `$this->redactor->redact()`
- [ ] `Redactor` lives in `Internal/` and is never referenced by consumers
- [ ] Sensitive keys list covers: pan, pan2, hpan, password, token, phone, pinfl, authorization, sender, recipient
- [ ] Redaction is recursive — nested arrays are also redacted
- [ ] Redacted value is always `'[REDACTED]'`
- [ ] Request ID generated by `RequestIdGenerator` using `bin2hex(random_bytes(16))`
- [ ] Request ID carried through request log, envelope, response validation, response log
- [ ] Consumer-provided request ID is accepted via Payload but stripped before provider call
- [ ] Transport failures log exception class name only — never raw exception message
- [ ] `NullLogger` used as fallback when no logger injected — never `Log::` facade
